  <name>Task 1: Add fetch_stats to database and on_stats handler</name>
  <files>lua/store/database/init.lua, lua/store/ui/store_modal/event_handlers.lua</files>
  <action>
**In `lua/store/database/init.lua`:**

1. Add the stats URL constant near the top (after the HTTP_HEADERS definition, following the same pattern as other constants):
   ```lua
   local STATS_URL = "https://store-nvim-telemetry.alex-popov-tech.workers.dev/stats?period=month"
   ```

2. Add a new public function `M.fetch_stats(callback)` before the `return M` line:
   ```lua
   ---Fetch plugin install stats from telemetry API
   ---@param callback fun(data: table|nil, error: string|nil) Callback with stats data or error
   function M.fetch_stats(callback)
     if not config.get().telemetry then
       callback(nil, nil) -- silently skip when telemetry disabled
       return
     end
     fetch_remote_json(STATS_URL, function(data, err)
       if err then
         logger.warn("Failed to fetch stats: " .. err)
         callback(nil, err)
         return
       end
       callback(data, nil)
     end)
   end
   ```

**In `lua/store/ui/store_modal/event_handlers.lua`:**

1. Add a new handler `M.on_stats(modal, data, err)` after the existing `on_installed_plugins` function:
   ```lua
   ---Handle stats data response from telemetry API fetch
   ---@param modal StoreModal The modal instance
   ---@param data table|nil Stats data from telemetry API
   ---@param err string|nil Error message if fetch failed
   function M.on_stats(modal, data, err)
     if err or not data or not data.stats then
       logger.warn("Stats fetch failed or empty: " .. tostring(err))
       return -- leave download_stats as initialized (nil)
     end
     local lookup = {}
     for _, entry in ipairs(data.stats) do
       lookup[entry.plugin_full_name] = entry.installs or 0
     end
     modal.state.download_stats = lookup
     logger.debug("Stats loaded: " .. vim.tbl_count(lookup) .. " plugins with download data")
   end
   ```

Note: on_stats does NOT trigger any re-render. The stats are passively available for the next sort operation. This follows the fire-and-forget pattern -- the data is there when the user chooses "Most Downloads".
